# Add tooltips to buttons and wheel segments

Add a reusable tooltip component that shows context text on hover for the Spin/Skip buttons, the four shop buttons, and the wheel wedges, positioned with a small offset above the target.

## Plan

1. **Create a reusable tooltip component** in `src/gameobjects/tooltip.ts`.
   - Accepts text content and an anchor game object (or a `Vec2`).
   - Draws a dark rounded background with light text, sized to fit the text.
   - Hidden by default; shown on hover and positioned slightly above the target.
   - Support an optional `anchor`/`offset` so a caller can request placement above the target (e.g., Spin/Skip buttons are fixed above their button).
   - Expose `show(text)`, `hide()`, `setTarget(obj)` helpers.

2. **Add tooltip support to `src/gameobjects/button.ts`**.
   - Add an optional `tooltip?: string` field to `AddButtonOptions`.
   - On hover, create/show a tooltip above the button; on hover end, hide it.
   - Reuse the existing `onHover`/`onHoverEnd` events so no duplicate listeners are needed.

3. **Wire tooltips in `src/gameobjects/shop.ts`**.
   - Pass descriptions for each shop button:
     - **Extra Spin**: cost/effect text, e.g., "Spend $2 to gain an extra spin this round"
     - **Upgrade Wheel**: cost/effect text, e.g., "Spend $5 to upgrade a positive wheel segment"
     - **Add Segment**: effect text, e.g., "Add a random positive or negative segment to the wheel"
     - **Continue**: action text, e.g., "Continue to the next round"
   - Keep the text derived from the `SHOP` constants so it stays in sync.

4. **Wire dynamic tooltips in `src/scenes/game.ts`**.
   - Pass tooltips to `Spin` and `Skip` that reflect current state:
     - Spin: "N spin(s) remaining" (or the chosen dynamic text).
     - Skip: "Skip this turn"
   - Update the tooltip text when `spinsRemaining` changes via the existing `updateSpinButton` or `updateUI` flow.

5. **Add a tooltip to the header score label in `src/gameobjects/header.ts`**.
   - Add an `area()` component to the score label so it can detect hover.
   - On hover, show a tooltip like "Score 100 points to clear the level" using the current `target` value.
   - Hide the tooltip when the hover ends.

5b. **Add a tooltip to the header money label in `src/gameobjects/header.ts`**.

- Add an `area()` component to the money label so it can detect hover.
- On hover, show a tooltip like "Money is used to buy things in the shop"
- Hide the tooltip when the hover ends.

6. **Add tooltips to `src/gameobjects/wheel.ts`**.
   - Compute the wheel segment under the mouse from angle and radius.
   - On hover, show the tooltip near the mouse/target with the segment's effect, e.g., "Score 50 points", "Lose 25 points", "Earn $5", "Pay $3".
   - Hide tooltip when the mouse leaves the wheel or when the wheel is spinning.

7. **Export and register the new component**.
   - Re-export `addTooltip` from `src/gameobjects/index.ts` if needed.

8. **Verify and run checks**.
   - `npm run lint:tsc` to catch TypeScript errors.
   - `npm run lint:fix` to auto-format and fix import order.
   - `npm run build` to ensure Vite builds cleanly.
